Case 02 · B2C Brand

B2C 브랜드 신규 트위터 채널 셋업 · 그로스

1주 만에 회사 자체 채널 팔로워 3,400 — 광고비 ₩0 오가닉 성장

Context

광고대행사는 캠페인마다 처음부터 다시 시작한다. 콘텐츠·청중·데이터가 전부 광고주 계정에 남는다.

결과적으로 대행사 자체 자산은 쌓이지 않는다.

Insight

광고대행사가 여러 광고주 제품을 홍보하면서도 대행사만의 정체성을 구축하는 자체 계정 을 만들면, 광고주가 바뀌어도 톤·청중은 대행사에 남는다.

그 청중에 광고주가 붙으려는 순간 단발 거래가 장기 관계 로 바뀌고, 검색·배너와 다른 새 광고 자리 가 만들어진다.

Action

  1. 페르소나 채널 운영 — '벼랑끝의 마케터' 캐릭터로 톤을 일관되게, 대행사 명의로 콘텐츠 발행.
  2. 광고 슬롯 자산화 — 광고 같지 않은 슬롯을 설계해 대행사 고유 자산 으로 포지셔닝, 광고주 유치.

02-2 플랫폼 분석 + 벤치마킹 → 페르소나 설계
Benchmarking
Step 3 — Benchmarking

위트성 트윗으로 감에 의존해서 팔로워를 늘리는 방식은 지속 불가능. 콘텐츠를 매일 짜내야 하는데 그 부담을 낮추려면, 플랫폼의 공감 회로 를 먼저 들여다보고 거기 맞춰 컨셉을 설계해야 한다.

트위터에서 가장 공감대가 큰 카테고리 = 마케팅·취업 정보를 다루는 직장인 페르소나. 범용적이라 도달이 넓고, 콘텐츠를 짜내기 쉬워 지속 가능하다는 판단.

  1. 플랫폼 공감 회로 확인 — 네임드 계정 리스트업
    마케터 · 직장인 카테고리에서 구독비율 높은 계정 선별. 왜 이 카테고리가 트위터에서 잘 먹히는가 패턴 파악.
  2. 공통 패턴 분석
    프로필 사진 톤 · 소개글 구조 · 핀고정 트윗 형식 · 게시 주기의 공통점 추출.
  3. 패턴 차용 + 본 계정 차별점 결합
    검증된 구조는 활용하되, '광고대행사 마케터'라는 차별 정체성 으로 직장인 페르소나의 확장형 포지션 확보.
Benchmark Examples

직장인 페르소나가 꾸준히 높은 공감과 인기를 얻는 패턴을 확인.

광고대행사 마케터는 직장인의 확장된 정체성 — 같은 공감 회로를 활용 가능하다고 판단.

'벼랑끝의 마케터' 컨셉 채택 — 콘텐츠 지속 생산 + 청중 공감 + 대행사 자체 자산 누적 세 가지를 한 번에.

02-4 유저 간 자연 발화 — 광고가 대화가 되는 메커니즘
Organic Diffusion

단순 증정 이벤트로 설계하지 않았다. 브릭샌드 메뉴 카탈로그를 노출하고 10 초 안에 먹고 싶은 맛 그리기 미션을 던졌더니 — 사람들이 직접 자기 손으로 메뉴를 그려서 응모하기 시작했다. 참여 행위가 곧 브랜드 학습으로 변하는, 예상하지 못한 결과.

트위터의 두 가지 특성 — 인용 RT 의 강한 오가닉 노출 + 친구 대화 같은 친밀한 댓글 스레드 — 위에서 광고비 ₩0 으로 브랜드 인지도가 자연스럽게 올라갔다. 광고를 광고로 인식하지 않는 환경에서 유저가 유저에게 말하는 자연 확산 구조 가 만들어진 셈.

Case 03 · 1-Person Build

Focus Flow — 특화 페르소나 PWA 그로스

혼자 개발한 앱이 2주 만에 ₩0 광고 · 채널 1 개로 360 명 커뮤니티가 되기까지

Context

기록을 좋아하는 사용자들이 쓸 도구 중, 감정 해소(일지) · 정보 보관(스크랩) · 동료 인증(바디더블링) 을 한 번에 담는 도구가 시장에 없었다.

각 기능을 따로 쓰면서 흩어진 맥락은 사용자의 지속성을 깎고 있었다. 직접 빌드해 풀기로 결정했다.

Insight

올인원 도구일수록 일반 시장에 알리기 어렵다. 특화 페르소나를 잡아야 한다.

시장 분석: 트위터에서 ADHD 콘텐츠가 강한 트래픽 + 여성 이용자 비중이 압도적.

활용 가능 자산: ADHD 콘텐츠로 자리 잡힌 트위터 계정 (팔로워 2,900+) → organic 도달 즉시 가능.

1차 타겟: ADHD 여성 트위터 사용자.

Action

  1. 채널 한 곳 · 페르소나 한 명에 자원 몰아 시작 — 트위터 단일 채널 + 운영 중인 계정으로 organic 도달
  2. 핵심 기능만 빌드 — 감정 일지 · 정보 스크랩 · 그룹 바디더블링. 결제 · 구독 · 일반인용 범용 기능 · 무거운 백엔드는 의도적으로 배제
  3. 1인 풀스택 빌드 — 제품 · 디자인 · 운영을 동시에 (Firebase + PWA)

Case 04 · TikTok Analytics

틱톡 영상 단위 데이터 분석 — 4 달만에 팔로워 4,800

평균 수치로 안 보이던 4 가지 비대칭 발견 → 콘텐츠 재설계로 월 평균 뷰 8 배 성장

Context

틱톡 기본 분석 도구는 평균 수치만 보여준다. 영상 하나씩 비교 분석할 외부 도구도 없다.

평가 기준 자체가 누락된 환경 — 어떤 콘텐츠가 진짜 효과 냈는지 모른 채 운영 중이었다.

Insight

영상 단위 데이터를 직접 들여다보니, 평균 수치로는 안 보이던 비대칭 4 가지가 드러났다.

  1. 조회수가 가장 많은 영상이 팔로워는 가장 적게 데려옴 — 조회수만 보면 평가가 틀어진다
  2. 가장 많은 시간 쏟은 카테고리 (밈, 시간 51 %) 가 전환율이 가장 낮음 — 시간 투자 ≠ 효과
  3. 같은 영상이라도 뷰만 1 위인 시간대 와 뷰+전환 1 위인 시간대 가 다름
  4. "짧을수록 좋다" 통념은 거짓 — 31~45초가 11~20초보다 팔로워 9 배 더 데려옴

Action

4 가지 비대칭을 그대로 4 가지 행동 변경으로 옮겼다.

  1. 평가 기준 — 조회수만 보던 단일 지표를 도달 + 전환 이중 지표 로 변경
  2. 카테고리 재배분 — 시간 가장 많이 쏟던 밈 (51 %) 줄이고 그림 강의 영상 중심 으로
  3. 업로드 시간 — 주말 20시 (뷰만) → 평일 22시 (뷰 + 전환)
  4. 영상 길이 — 11~20초 위주 → 31~45초 중심

04-0 왜 직접 분석 대시보드를 만들었나
WHY
현상 — 틱톡 기본 분석 도구는
영상 한 개씩 따로 데이터가 흩어져 보임 — 조회수 226K · 시청 시간 716 h · 평균 시청 11.27 s · 새 팔로워 110 …

여러 영상을 한 화면에 모아 비교 해서 어떤 컨셉이 진짜 효과 냈는지 판단할 외부 도구가 없었다.
해결 — 직접 빌드한 분석 환경
모든 영상 데이터를 크롤링 해서 한 곳에 모으고, 비교 분석 가능한 대시보드 를 직접 만들었다.
  • 영상 단위 조회수 · 시청률 · 신규 팔로워 · 전환율 한 표에
  • 카테고리별 · 시간대별 · 길이별 비대칭 발견 가능
  • 이중 KPI (도달 + 전환) 프레임으로 평가
평균 수치 뒤에 숨은 비대칭 이 이 환경에서 드러났다.

웃긴 영상은 17.3 만 뷰 인데 신규 팔로워는 87 명 (전환률 0.05 %). 반면 그림 작업 영상은 비슷한 조회수 (11.6 만 뷰) 인데 신규 팔로워 320 명 — 전환률 0.28 %, 약 5.6 배 차이.

그래서 → 콘텐츠 방향성을 그림 강의 컨셉으로 개편. 지속적으로 팔로워 파이를 늘리려면 조회수가 아닌 전환률 기준 으로 콘텐츠 재설계가 필요했다.

틱톡 알고리즘과 연령층이 보는 건 영상 퀄리티가 아니라 컨셉이 시청자에게 즉시 닿는가. 단순 채색 팁이 "이거 나도 따라할 수 있겠다" 감각을 만들고, 그게 팔로우로 직결됐다.

그래서 → 콘텐츠 공수 ≠ 결과 비례. 들인 시간보다 컨셉 적합도 가 우선이라는 걸 실측치로 학습 — 이후 영상 제작 비용을 90 % 줄이면서 성과는 오히려 끌어올렸다.

Case 05 · AI Content Automation

AI 자동화 콘텐츠 운영 — 잘 된 글 분석해서 따라하기 + 매체별 말투 학습

감으로 안 쓴다 — 잘 된 형식을 AI 가 배워서 더 잘 터지는 글 자동 생성

Context

콘텐츠 1 글 작성 = 정통 2 시간. 게다가 플랫폼별 톤·주제가 달라 1 인이 여러 채널 동시 운영 어려움.

추가 문제 — 기존 콘텐츠 발행은 감으로 쓰니까 히트 확률이 낮다. 사람이 일일이 인기 형식 분석할 시간도 부족.

Insight

AI 는 단순 생산성 도구가 아니라 형식 학습 도구. 잘 된 콘텐츠의 형식·톤·구조를 AI 에 학습시키면 터질 확률이 오른다.

  1. 플랫폼별 최적 컨셉 분리 — 블로그 = 영어 공부 + 자유글 (검색 기반, 누적), 트위터 = 자기개발 (바이럴, 단일 임팩트)
  2. 플랫폼별 말투까지 학습 — 같은 주제도 블로그/트위터 톤이 다름
  3. 잘 된 콘텐츠 형식을 분석해서 따라하기 — AI 프롬프트에 형식 학습시키기
  4. 발행 후 데이터 측정 → 잘 되는 거 다시 형식으로 환원 (피드백 사이클)

Action

두 가지 채널 동시 운영 — 보완 관계의 양극 전략을 1 인이 실행.

  1. Viral 채널 (트위터) — 자기개발 키워드 검색 → 상위 노출 글 작성자 식별 → 그들의 인기 트윗을 AI 학습 데이터로. 알고리즘이 추천하는 비슷한 계정 chain 을 따라가며 학습 데이터 풀 확장.
  2. Long-tail 채널 (블로그) — 플랫폼 자체 분석 도구 로 검색량 대비 발행량 낮은 키워드 발굴 + AI 작성, 34 글 발행
  3. 잘 된 콘텐츠 형식 을 AI 학습 → 매 발행 마다 히트 확률 상승 (피드백 사이클)

AI 는 기업에게 필수가 됐다 — 안 쓰면 경쟁력이 빠르게 벌어진다. 다만 그 방식을 바꿔야 한다.

AI 가 발달할수록 사람들은 오히려 아날로그 감성을 더 찾게 된다. 그런데 AI 가 그 감성까지 흉내낼 수 있으니, 결국 진짜와 가짜를 구별하기 어려워진다.

그래서 AI 마케팅의 진짜 방향은 — 효율성만 쫓아 무조건 AI 에 맡기는 게 아니라, 접근 방식과 회색지대를 정확히 가늠하는 것. 어디까지 AI 가 맡고 어디까지 사람이 다듬을지를 판단할 수 있어야 한다.

05-2 AI 트윗생성기 실측 — 외로움 약한 연결 3 타래 / 누적 15.7만 노출
Live Result

AI 트윗생성기 가 학술 근거 (Granovetter 1973, Sandstrom & Dunn 2014) 기반으로 만든 3 타래 트윗. 자기개발 + 사회심리 주제, 자연스러운 톤으로 작성.

학습 데이터 출처 — 사람의 결을 어떻게 모았나
시드 계정 — @4_wonderfullife (책 인용 · 손그림 · 사색 톤)
알고리즘 추천 — 결 비슷한 계정 풀 확장

시드 계정 한 명에서 시작. 트위터 알고리즘이 띄워주는 "내가 좋아할 만한 콘텐츠" 추천으로 결이 같은 계정 풀을 확장. 이 사람들의 문장 호흡 · 줄바꿈 · 감정 어휘를 학습 데이터로 묶어서 트윗생성기에 먹임. 키워드 카피가 아니라 사람의 결을 알고리즘이 대신 모아 준 셈.

AI 가 인간 심리를 인간의 사색적 말투로 흉내내서 쓴 트윗 — 사람들이 더 진짜로 받아들였다. 첫 트윗 단독 138K 노출, 3 타래 누적 15.7만 노출.

역설 — 사람들이 "AI 콘텐츠 가 이질감 있어서 싫다" 고 말하면서도, AI 가 사람보다 더 사색적이고 정제된 톤으로 쓰면 진짜 사람 글로 받아들이고 더 깊게 반응한다. 결국 차별점은 AI 사용 여부 가 아니라 얼마나 사람의 결을 정교하게 학습시켰는가.

05-3 AI 가 쓴 독백 블로그 — 3 분 작성 / 오가닉 누적 800 회 가까이
